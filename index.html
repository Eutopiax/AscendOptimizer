Abstract

NPU Optimization LLM Agent Auto-tuning

AscendC operator optimization on Huawei Ascend neural processing units (NPUs) faces a two-fold knowledge bottleneck: unlike the CUDA ecosystem, there are few public reference implementations to learn from, and performance hinges on a coupled two-part artifact‚Äîa host-side tiling program that orchestrates data movement and a kernel program that schedules and pipelines instructions.

We present AscendOptimizer, an episodic agent that bootstraps this missing expertise by turning execution into experience. On the host side, AscendOptimizer performs profiling-in-the-loop evolutionary search to discover valid and high-performing tiling and data-movement configurations directly from hardware feedback. On the kernel side, it mines transferable optimization motifs by rewinding optimized kernels‚Äîsystematically de-optimizing them to synthesize instructive "bad-to-good" trajectories‚Äîand distills these motifs into a retrievable experience bank for guided rewriting.

Key Results: On a benchmark of 133 real AscendC operators, AscendOptimizer achieves a 1.34√ó geometric-mean speedup over the open-source baseline, with 51.88% of operators outperforming their references, outperforming strong agent and search baselines.

Introduction

As the parameter scale of large language models (LLMs) advances towards the trillion level, the supply of computational resources has become a core factor constraining the development of artificial intelligence. Operators, as the atomic execution units of computation graphs, directly determine the training throughput and online inference response latency of models.

The Knowledge Scarcity Problem

Unlike the NVIDIA GPU ecosystem which has established a mature automated optimization toolchain (TVM, Ansor, LLM-driven optimization), transferring automation to domain-specific accelerators (DSAs) remains challenging. We specifically target the Huawei Ascend NPU, which uses the Da Vinci architecture with an explicitly managed memory hierarchy.

Knowledge Bottleneck

Few public reference implementations exist for AscendC, unlike the abundant CUDA ecosystem.

Two-Part Artifact

Performance depends on both host-side tiling program and device-side kernel program.

LLM Generalization Gap

MultiKernelBench shows CUDA Pass@1 reaches 44-52%, while AscendC drops below 2.1%.

Our Insight: When external data are insufficient, we can bootstrap experience internally by exploiting the structured nature of code. AscendOptimizer alternates between optimizing tiling ùíØ with the kernel fixed, and optimizing the kernel Œ∫ with the tiling fixed, so improvements in one block reshape the feasible and high-performing region of the other.

Alternating Optimization Framework

AscendOptimizer optimizes the host-side tiling function ùíØ and the device-side kernel code Œ∫ in two stages through a block coordinate descent procedure:

Stage I: Evolutionary-Guided Program Search
  • Models tiling optimization as a program search problem
  • Leverages LLMs to perform evolutionary search within the function space
  • Implicitly learns hardware constraints via Hardware-in-the-Loop (HIL) feedback
  • Rapidly converges to high-quality tiling strategies within the implicit feasible region
Stage II: Optimization-Rewind Experience Bootstrapping
  • Deliberately rewinds (removes) optimizations in seed implementations
  • Constructs an optimization pattern library from "bad-to-good" trajectories
  • Distills motifs into a retrievable experience bank
  • Enables retrieval-augmented kernel optimization under hardware feedback

Key Capabilities

Hardware-in-the-Loop

Uses actual hardware profiling feedback to guide the search, ensuring discovered configurations are truly performant.

Experience Bootstrapping

Automatically constructs optimization pattern library via "optimization rewind" without manual expert annotation.

Alternating Optimization

Block coordinate descent over tiling and kernel space, reshaping feasible regions iteratively.

Retrievable Experience Bank

Distills optimization patterns into machine-consumable form for RAG-based LLM rewriting.
